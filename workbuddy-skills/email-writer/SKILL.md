# email-writer

## 概述

将用户的中文要点自动转为专业英文邮件，直接展示邮件文本供复制粘贴。

## 触发条件

当用户输入包含以下关键词时立即触发本技能：
- "写邮件"/"写封邮件"/"帮我写邮件"
- "发邮件"
- "英文邮件"/"英语邮件"
- "给领导/老板/leader 写邮件"
- "邮件回复"/"回邮件"

## 工作流程

### Step 1: 确认邮件要素

主动向用户确认以下信息（如果用户已提供则跳过）：
1. **主题**：邮件主题（中文即可，翻译成英文）
2. **正文要点**：用户要表达的核心内容
3. **CC**：抄送人（可选）

### Step 2: 生成英文邮件

根据用户输入，按以下规范生成英文邮件：

**语气风格**：严格遵循 `references/tone-guide.md` 中的风格指南：
- 温暖谦逊、礼貌专业
- 先感谢再提需求
- 支持赋能型语气
- 符合外企（外资企业）商务规范

**段落结构**：
1. **开头**：直接进入正题，**不要使用** "I hope this message finds you well." 等套话问候，直接用 "I wanted to..." / "I'd like to..." 开头
2. **核心内容**：按逻辑分段，每段一个主题，使用 "I'd like to..." / "I wanted to provide an update on..." 等委婉表达
3. **结尾**：使用公司惯用语收尾 "Please don't hesitate to let me know if any further suggestions. Thanks."
4. **署名**：Regards,  
   Spring

**技术处理**：
- 中文要点 → 翻译并组织成流畅的英文段落
- 核心内容优先使用 **bullet points** 呈现，更清晰
- 保持段落之间有空行，方便阅读
- 署名固定为 **Spring**

### Step 3: 展示邮件文本供复制

1. 向用户展示完整的英文邮件文本供审阅
2. 询问用户是否需要调整

**输出格式**：
```
📧 邮件预览（可审阅修改后让我重新生成）：

Subject: [邮件主题]

---
[邮件正文]
---

请复制粘贴到 Outlook 发送。
```

### Step 4: 用户确认

用户确认邮件内容无误后，最终输出邮件文本。

## 注意事项

- 邮件中的中文姓名保持原文，不翻译
- 对于包含表格或复杂格式的邮件，建议生成纯文本版本并告知用户手动格式化
- 开头避免套话问候语
- 结尾保持简洁，去掉冗余表达
